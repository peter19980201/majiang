# 日麻计分助手 — 策划文档

## 1. 项目概述

**项目名称**：日麻计分助手
**平台**：微信小程序
**目标用户**：线下日麻玩家（有一定基础）
**核心价值**：快速准确地完成符数、番数、点数计算与分账，取代查表和心算

### 1.1 模式规划

- **单机模式（优先开发）**：一台手机充当记分台，所有人围着看
- **多人房间模式（后续开发）**：通过房间号加入同一对局，实时同步点棒变化

### 1.2 规则基准

- 以日本麻将联盟标准规则为基准
- 支持赤ドラ（赤宝牌）开关
- 默认半庄战，可选东风战

## 2. 功能规格

### 2.1 核心计算引擎

#### 2.1.1 手牌拆解器

- 输入：14 张牌（含和了牌）+ 副露信息
- 输出：所有合法拆解方式（面子 + 雀头的组合）
- 特殊牌型：七对子（7 个对子）、国士无双（13 幺）单独处理
- 算法：递归回溯法

#### 2.1.2 役种判定器

完整役种表见 [第 5 章](#5-役种表)。

判定所需上下文：
- 手牌构成（面子、雀头、副露）
- 门前（メンゼン）/ 非门前
- 和牌方式：自摸 / 荣和
- 和了牌具体是哪张
- 场风（東/南）、自风（東/南/西/北）
- 立直状态（普通立直 / W立直）
- 一发（イッパツ）
- 海底（ハイテイ）/ 河底（ホウテイ）
- 岭上开花（リンシャンカイホウ）
- 抢杠（チャンカン）
- 天和 / 地和

#### 2.1.3 符数计算器

```
基本符（底符）：
  - 门前荣和：30 符
  - 门前自摸：20 符
  - 非门前荣和：30 符（副露）
  - 非门前自摸：20 符（副露）

面子符：
  ┌─────────┬──────┬──────┐
  │         │ 明刻 │ 暗刻 │
  ├─────────┼──────┼──────┤
  │ 中张牌  │  2   │  4   │
  │ 幺九牌  │  4   │  8   │
  ├─────────┼──────┼──────┤
  │         │ 明杠 │ 暗杠 │
  ├─────────┼──────┼──────┤
  │ 中张牌  │  8   │  16  │
  │ 幺九牌  │  16  │  32  │
  └─────────┴──────┴──────┘
  ※ 顺子 = 0 符

雀头符：
  - 役牌（场风/自风/三元牌）：2 符
  - 连风牌（场风且自风）：2 符（部分规则 4 符，本项目取 2 符）
  - 其他：0 符

和牌方式符：
  - 边张/嵌张/单骑：2 符
  - 双面/双碰：0 符

合计 → 向上取整到 10 的倍数
```

特殊情况：
- 平和自摸 = 20 符（不取整）
- 七对子 = 固定 25 符
- 非门前平和荣和 = 30 符

#### 2.1.4 番数汇总器

- 累加所有成立役的番数
- 处理食下がり（副露降番）：部分役副露时番数 -1
- 加算宝牌（ドラ）、里ドラ、赤ドラ的番数（每张 +1 番）

#### 2.1.5 点数计算器

```
基本点 = 符 × 2^(番数+2)

满贯规则：
  - 3番70符 / 4番30符 以上 → 满贯（マンガン）
  - 6-7 番 → 跳满（ハネマン）
  - 8-10 番 → 倍满（バイマン）
  - 11-12 番 → 三倍满（サンバイマン）
  - 13 番以上 → 累计役满（数え役満）
  - 役满 → 役满

点数表（子家荣和时 = 基本点 × 4，向上取整到 100）：
  - 满贯：8000
  - 跳满：12000
  - 倍满：16000
  - 三倍满：24000
  - 役满：32000

亲家（庄家）= 上述 × 1.5
```

#### 2.1.6 分账计算

```
荣和（ロン）：
  - 放铳者支付全部点数

自摸（ツモ）子家和：
  - 其他子家各付：基本点 × 1（向上百位取整）
  - 亲家付：基本点 × 2（向上百位取整）

自摸（ツモ）亲家和：
  - 其他三家各付：基本点 × 2（向上百位取整）
```

#### 2.1.7 最优拆解选择

- 当一副手牌有多种合法拆解时，每种拆解分别计算点数
- 自动选取得点最高的拆解作为结果
- 结果页展示最优拆解的役种和得点明细

### 2.2 输入交互

#### 2.2.1 图形选牌器

- 展示所有 34 种牌面图形（万子 1-9、筒子 1-9、索子 1-9、字牌 7 种）
- 每种牌显示剩余可选数量（最多 4 张，赤牌另算）
- 分区输入：
  - **手牌区**：门前手牌（暗牌）
  - **副露区**：吃/碰/明杠/暗杠
  - **和了牌**：最后获得的那张牌
- 操作：点击添加，长按或再次点击移除
- 实时校验：牌数是否正确（手牌 + 副露 × 3 + 和了牌 = 14）

#### 2.2.2 副露输入

每组副露需要标记类型：
- **吃（チー）**：3 张顺子，标记来源方向（上家）
- **碰（ポン）**：3 张刻子
- **明杠（ダイミンカン / カカン）**：4 张，从他家获得
- **暗杠（アンカン）**：4 张，自己摸齐

副露数量：0-4 组

#### 2.2.3 场况设置

- 场风选择：東 / 南
- 自风选择：東 / 南 / 西 / 北
- 和牌方式：自摸 / 荣和
- 荣和时：放铳者是谁（影响分账，通过座位选择）

#### 2.2.4 附加状态

以下为开关/勾选项：
- 立直（リーチ）
- 双立直（ダブルリーチ）
- 一发（イッパツ）
- 海底摸月（ハイテイ）
- 河底捞鱼（ホウテイ）
- 岭上开花（リンシャンカイホウ）
- 抢杠（チャンカン）
- 天和（テンホウ）— 仅亲家第一巡自摸
- 地和（チーホウ）— 仅子家第一巡自摸

#### 2.2.5 宝牌输入

- 表ドラ指示牌：最多 4 张（支持翻杠ドラ）
- 里ドラ指示牌：最多 4 张（仅立直时有效）
- 赤ドラ开关：赤五万 / 赤五筒 / 赤五索 各 0-1 张
- ドラ判定逻辑：指示牌的下一张为ドラ（字牌循环：東→南→西→北→東、白→发→中→白）

### 2.5 对局记分板

#### 2.5.1 对局设置

- 对局类型：东风战（東風戦）/ 半庄战（半荘戦）
- 起始点棒：25000 点（可自定义）
- 返还点：30000 点（可自定义）
- 顺位马：10-20 / 10-30 / 5-10 等（可自定义）
- 四位玩家昵称

#### 2.5.2 对局进行

- 显示当前局数（例：東1局 0本場）
- 四家当前点棒一目了然
- 每局结束时录入结果：
  - 和了：选择和了者 → 进入手牌输入 → 自动计算并更新点棒
  - 流局：选择听牌者 → 自动计算不听罚符（3000 点分配）
  - 连庄/轮庄自动处理
- 本场棒：每本场和了额外 +300 点（自摸每家 +100）
- 立直棒：1000 点，由和了者回收；流局保留到下局

#### 2.5.3 对局结算

- 最终点棒排名
- 顺位马计算
- 素点（与返还点的差值 ÷ 1000）

#### 2.5.4 历史记录

- 每局详细记录（和了者、放铳者、役种、点数变动）
- 对局历史存储（wx.setStorageSync）
- 支持查看往期对局

## 3. 数据结构设计

### 3.1 牌的编码

```javascript
// 牌种编码：0-33
// 万子: 0-8  (一万~九万)
// 筒子: 9-17 (一筒~九筒)
// 索子: 18-26 (一索~九索)
// 字牌: 27-33 (東南西北白发中)

// 牌对象
{
  id: 4,        // 牌种 ID (0-33)
  red: false    // 是否为赤牌
}

// 幺九牌判定: id % 9 === 0 || id % 9 === 8 || id >= 27
// 中张牌判定: id < 27 && id % 9 !== 0 && id % 9 !== 8
```

### 3.2 副露结构

```javascript
{
  type: 'chi' | 'pon' | 'minkan' | 'ankan' | 'kakan',
```

### 7.1 项目结构（建议）

```
miniprogram/
├── app.js / app.json / app.wxss
├── utils/
│   ├── tiles.js          牌的编码与工具函数
│   ├── decompose.js      手牌拆解算法
│   ├── yaku.js           役种判定
│   ├── fu.js             符数计算
│   ├── score.js          番数/点数/分账计算
│   └── dataLayer.js      数据管理层（后续支持云端切换）
├── pages/
│   ├── index/
│   ├── calculator/
│   ├── game/
│   └── reference/
├── components/
│   ├── tile-picker/      选牌器组件
│   ├── tile-display/     牌面展示组件
│   └── score-board/      记分板组件
└── static/
    └── tiles/            牌面素材
```

### 7.2 注意事项

- 拆解算法需要处理好性能，14 张牌的拆解组合数有限，递归不会爆栈
- 役种判定顺序：先判定役满（满足则跳过普通役），再判定普通役
- 七对子和国士无双需要独立的拆解路径
- 符数计算在平和自摸时有特殊规则（固定 20 符）
- 连风牌雀头的符数各地规则不同，需明确选取一种
- 点数取整方向：向上取整到 100
